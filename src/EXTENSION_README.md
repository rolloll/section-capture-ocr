# Section Capture & OCR — Chrome 확장프로그램

웹 페이지에서 원하는 영역(섹션)을 지정해 **이미지(PNG/JPG)로 저장**하고,
그 이미지를 **OCR로 읽어 TXT 파일로 저장**하는 확장프로그램입니다.

OCR은 [Tesseract.js](https://github.com/naptha/tesseract.js)를 사용하며,
언어 데이터(**중국어 간체 / 한국어 / 영어**)가 확장에 포함되어 있어
**인터넷 연결 없이도 동작**합니다. 기본 인식 언어는 중국어 간체이며,
마지막에 선택한 언어 조합이 다음 실행에도 유지됩니다.

## 설치 방법 (개발자 모드 로드)

1. Chrome 주소창에 `chrome://extensions` 입력
2. 우측 상단 **개발자 모드** 켜기
3. **압축해제된 확장 프로그램을 로드합니다** 클릭
4. 이 폴더(`section-capture-ocr`)를 선택
5. 툴바에 📸 아이콘이 나타나면 설치 완료

## 사용 방법

### 1) 영역 캡처 → 이미지 저장

1. 캡처할 웹 페이지에서 확장 아이콘 클릭
2. 두 가지 모드 중 선택:
   - **✂️ 영역 드래그 캡처** — 마우스로 원하는 영역을 직접 드래그.
     **드래그 중 화면 아래(또는 위) 가장자리에 마우스를 대면 자동으로 스크롤**되면서
     한 화면을 넘는 범위까지 계속 선택할 수 있음 (마우스 휠 스크롤도 가능).
     선택 크기 표시에 `(스크롤 캡처)`가 붙으면 여러 장을 자동으로 이어붙여 저장함
   - **🎯 요소(섹션) 선택 캡처** — 마우스를 올리면 해당 섹션(요소)이 주황색으로
     강조되고, 클릭하면 그 섹션 전체가 캡처됨 (Listly의 요소 선택 방식과 유사).
     **화면보다 긴 섹션도 자동 스크롤하며 전체가 캡처됨**
   - **📖 연속 캡처 (뷰어용)** — 캡처할 **본문 섹션 위에 마우스를 올려 강조되면
     클릭**으로 선택(드래그 불필요). 선택 직후 **몇 구간(화)까지 캡처할지** 입력창이
     뜨며, 숫자를 넣으면 그만큼만, 비워두면 끝까지(최대 300화면) 캡처함.
   - **jjwxc 화 단위 연속 캡처** — jjwxc 챕터 페이지(URL에 novelid/chapterid 포함)에서
     연속 캡처를 시작하면 "몇 화까지 캡처할까요?" 입력창이 뜨고, 숫자를 넣으면
     **chapterid를 1씩 올려가며 다음 화 페이지로 자동 이동**해 같은 섹션 영역을
     이어서 캡처함 (화마다 별도 이미지로 저장). 페이지 이동 중에는 탭을 그대로 두세요. 좌우로 넘겨보는 뷰어에서는 **자동으로 페이지를
     넘기면서** 각 페이지를 세로로 이어붙임. 내용이 더 바뀌지 않으면(마지막 페이지)
     자동 종료되며, ESC 키나 우하단 「캡처 중지」 버튼으로 원하는 지점에서 멈출 수 있음.
     (→ 방향키로 넘기는 뷰어와 클릭으로 넘기는 뷰어 모두 자동 대응)
     **카카오페이지 웹 뷰어처럼 스크롤 방식이면 자동 감지**해서 연속 캡처로 전환:
     현재 구간을 끝까지 캡처한 뒤 **[다음]/아래 화살표 버튼을 자동으로 눌러**
     다음 구간을 이어서 캡처함 (최대 300화면, 구간마다 별도 이미지로 저장).
     ESC나 「캡처 중지」 버튼으로 언제든 멈출 수 있음.
   - 캡처들은 결과 탭 상단의 **캡처 목록**에 쌓이며(최대 240개), 선택/삭제 가능.
     **📚 전체 일괄 OCR** 버튼으로 저장된 모든 캡처를 순서대로 인식해
     하나의 텍스트로 합칠 수 있음.
3. 파일 저장(PNG/JPG/TXT) 시 **파일 이름을 지정하는 팝업**이 뜹니다.
   기본값은 자동 생성 이름이며, 취소하면 저장하지 않습니다. 기본 이름 형식(축약형):
   - 카카오페이지: `kk뷰어번호_년월일_시분(_p부번호)` — 예: `kk66498342_260711_1843_p5`
   - jjwxc: `jj소설번호_화번호_년월일_시분` — 예: `jj8650107_21_260711_1908`
   - 起点(qdmm/qidian): `qd책번호_화번호_년월일_시분`
   - 기타 사이트: `cap_년월일_시분` 또는 `호스트명_년월일_시분`
   - **TXT 파일은 텍스트 출처에 따라 접미사가 붙습니다**:
     로컬 OCR만 = 접미사 없음 / 로컬 OCR+AI 교정 = `_fix` / AI 비전 OCR = `_AI`
     — 예: `jj8650107_21_260711_1908_AI.txt`
4. **📦 일괄 다운로드**: 캡처 이미지 패널 아래의 "일괄 다운로드"를 펼쳐
   원하는 캡처들을 체크하고, 이미지(PNG/JPG)와 텍스트(OCR) 중 받을 것을 골라
   **한 번에 저장**할 수 있습니다. 텍스트를 선택하면 캡처마다 OCR이 자동 실행되어
   각각 .txt 파일로 저장됩니다. 파일명은 위의 자동 형식이 적용됩니다.
   - ESC 키로 언제든 취소 가능
3. 한 화면을 넘는 범위는 자동으로 스크롤하며 나눠 찍은 뒤 한 장으로 이어붙임
   (진행률이 표시되며, 캡처 중에는 페이지를 조작하지 마세요)
4. 완료되면 **결과 탭**이 자동으로 열림 — **PNG로 저장** 또는 **JPG로 저장** 클릭 (JPG 품질 조절 가능)

### 2) OCR → TXT 저장

1. 결과 탭 오른쪽 패널에서 인식할 언어 선택 (中文(간체) / 한국어 / English)
   — **캡처한 페이지 기준으로 자동 선택**됩니다: 한국 사이트(카카오페이지, 네이버,
   리디 등 또는 `.kr` 도메인)는 한국어, 중국 사이트(jjwxc, 起点 등 또는 `.cn`)는
   중국어가 자동으로 켜지고, 도메인으로 알 수 없으면 페이지 제목의 문자(한글/한자)로
   판별합니다. 판별이 안 되면 마지막 선택이 유지되며, 언제든 손으로 바꿀 수 있습니다.
2. **🔍 OCR 실행** 클릭 — 진행률이 표시되며 완료되면 텍스트가 나타남
3. 필요하면 텍스트를 직접 수정한 뒤 **TXT로 저장** 클릭
   (UTF-8(BOM) 인코딩이라 메모장에서 한글이 깨지지 않음)
4. **복사** 버튼으로 클립보드 복사도 가능

저장 파일명은 `capture_YYYY-MM-DD_HH-mm-ss.png / .jpg / .txt` 형식입니다.

## 폴더 구조

```
section-capture-ocr/
├── manifest.json      # 확장 설정 (Manifest V3)
├── popup.html/js      # 툴바 팝업 (모드 선택)
├── content.js         # 페이지 내 영역 선택 오버레이
├── background.js      # 화면 캡처 + 타일 스트리밍 저장
├── tile-store.js      # IndexedDB 타일 Blob 저장소 (SCOStore)
├── viewer.html/js     # 결과 화면 (이미지 저장 / OCR / TXT 저장)
├── site-adapters.js   # 사이트별 어댑터 (jjwxc/qidian/kakao)
├── xlsx-lite.js       # 용어집 XLSX/CSV 파서
├── fonts/             # 내장 글꼴 (WOFF2)
├── icons/             # 확장 아이콘
├── lib/               # Tesseract.js 라이브러리 + WASM 엔진
└── tessdata/          # OCR 언어 데이터 (chi_sim, kor, eng)
```

## v2.18.2 (타일/OCR/배치 무결성 감사 반영)

- **타일 복원 fail-closed**: 저장된 타일 복원 실패 시 빈 캔버스로 대체하지 않고 중단 —
  같은 위치 재시도가 완성 타일을 조용히 덮어쓰던 문제 차단. (BLOCKER-001)
- **OCR 병합 정정**: 같은 소스 내부의 인접 구간은 줄바꿈으로만 잇고, 서로 다른 타일 경계
  (실제 이미지가 겹치는 곳)에서만 중복 문장을 병합 — 정당한 반복 줄이 삭제되던 문제 수정. (BLOCKER-002)
- **일괄 처리 작품 오염 차단**: 일괄 OCR·다운로드가 각 캡처의 '자기 작품' 규칙/용어집으로
  후처리하도록 변경(전역 선택 작품이 다른 작품 캡처에 잘못 적용되던 문제). 혼합 배치에서는
  인물 자동 수집 비활성화. (BLOCKER-003)
- 작품 프로필 매칭을 사이트별 구조 키(`jjwxc:`/`qidian:`/`kakao:`)로 확장(기존 `novelid=`
  호환 유지). (HIGH-001)
- OpenAI `message.refusal` / Anthropic `stop_reason:"refusal"` 구조화 거부를 오류로 처리. (HIGH-002)
- Sonnet 5·Opus 4.8 요청에서 `temperature` 제거(예측 가능한 400→재시도 제거). (HIGH-003)
- 빈 구간 판정을 '거의 완전 균일할 때만'으로 더 보수화(희소·흐린 글자 보호). (HIGH-004)
- 고아 타일 정리를 단일 큐로 직렬화. (HIGH-005)
- endCapture 실패 시 이 캡처의 타일을 롤백하고 job 정리. (MEDIUM-001)
- 연속 캡처에서 '마지막 세그먼트 직후 중지' 시 완성본 저장을 막아 재개 중복 방지. (MEDIUM-002)
- 캡처 전환 중 늦게 끝난 OCR/비전 상태 복원이 다른 캡처를 덮어쓰지 않도록 가드. (MEDIUM-003)
- 비전 통계를 구간 상태(로컬 대체/복구 실패/AI)로 정확히 집계. (MEDIUM-004)
- 비전 진행 상태를 5구간마다 체크포인트 저장. (MEDIUM-005)
- Anthropic 비용 추정가 정정(Opus 4.8 $5/$25, Sonnet 5 도입가 $2/$10). (MEDIUM-006)
- 일괄 묶음 키를 사이트 계열 기준으로 정규화(www./m. 서브도메인 통합). (LOW-001)
- 배포 zip 경로 구분자를 POSIX(`/`)로. (LOW-002)

## v2.18.1 (자동 묶음이 예전 캡처에 적용되지 않던 문제 수정)

- v2.18.0에서 추가한 자동 묶음이 **그 이전에 저장된 캡처에는 전혀 적용되지 않던 결함

  각 캡처의 전체 데이터(pageUrl은 처음부터 저장돼 있었음)에서 값을 읽어와 인덱스를
  1회 보정하도록 수정 — 이제 예전에 캡처한 항목도 자동 묶음이 적용된다.

  SIL OFL 1.1 오픈폰트 라이선스로 재배포 가능)

  순서 설정으로 바꾸면서 이 접미사가 순서 설정에 나타나지 않게 됐던 것을, 다른 토큰과
  동일하게 순서를 정할 수 있는 「텍스트 출처」 항목으로 다시 추가(기본 순서의 맨 뒤에
  위치). 로컬 OCR만=접미사 없음 / 로컬+AI 교정=`fix` / AI 비전 OCR=`AI`.

## 참고 / 제한 사항

- 한 화면을 넘는 범위는 자동 스크롤로 나눠 찍어 이어붙입니다. 이때 화면에 고정된
  요소(상단 고정 메뉴 등)는 중복으로 찍히지 않도록 두 번째 장부터 잠시 숨겨집니다.
- 세로로 매우 긴 캡처는 안전 한도(약 32,000px)까지만 저장됩니다.
- 선택 영역이 **화면에 고정된 패널**(댓글창, 채팅창 등 내부에서 자체 스크롤되는 창)을
  덮고 있으면, 페이지 대신 **패널 내부를 자동 스크롤**하며 현재 위치부터 끝까지
  이어붙여 캡처합니다. (진행 표시에 "내부 스크롤"이 표시됨)
- 고정 패널이 아닌 일반 본문 속의 내부 스크롤 박스는 지원되지 않습니다.
- 가로 폭은 한 화면 폭까지 지원합니다 (세로 스크롤 캡처 전용).
- `chrome://` 등 브라우저 내부 페이지에서는 보안상 캡처할 수 없습니다.
- OCR 정확도는 이미지 해상도에 비례합니다. 글자가 작으면
  페이지를 확대(Ctrl + `+`)한 뒤 캡처하면 인식률이 올라갑니다.
- 첫 OCR 실행 시 엔진과 언어 데이터를 로딩하느라 몇 초 걸릴 수 있습니다.
- 중국어 간체·한국어 모두 **고정밀(tessdata_best) 언어 데이터**를 내장하고 있으며,
  OCR 실행 시 이미지를 자동으로 2~4배 확대하고 회색조+명암 정규화
  (다크 테마 자동 반전)를 거친 뒤 인식합니다(**고정밀 보정** 옵션, 기본 켜짐).
  획이 조밀한 한자가 뭉개지지 않도록 흑백 이진화는 쓰지 않습니다.
  소설 본문처럼 글만 있는 화면은 **본문 모드**(기본 켜짐)가 더 정확하고,
  표/버튼이 섞인 화면이면 꺼 보세요.
- **부호 통일**(기본 켜짐): 영어에도 있는 문장부호(？ ！ ： ； ，
  “” ‘’ （） ％ 등)를 반각 영문 기호(? ! : ; , " ' ( ) %)로 통일합니다.
  OCR이 전각/반각·둥근따옴표를 오락가락 인식해서 생기는 불일치가 사라집니다.
  중국어 고유 부호(。 、 《》 【】 「」)는 그대로 유지됩니다.
  말줄임표(…)는 `...`로 통일됩니다.
- 그래도 인식률이 낮으면 페이지를 125~150%로 확대(Ctrl + `+`)한 뒤 캡처하세요.
  글자가 클수록 정확합니다.
- 여러 언어를 동시에 켜면 속도가 느려지고 오인식이 늘 수 있으니,
  중국어 본문만 읽을 때는 中文(간체) 하나만 선택하는 것이 가장 정확합니다.
- 긴 캡처는 문단 사이 여백을 찾아 구간별로 나눠 인식한 뒤 이어붙입니다
  (진행 표시에 `n/m 구간`으로 표시됨).
- jjwxc 본문에 삽입되는 워터마크(`@无限好文，尽在晋江文学城`)는 OCR 변형 포함
  자동으로 제거됩니다.
- **괄호 주석 정리**(기본 켜짐): 한국어 번역본에서 괄호 안 한자 병기가 OCR로 깨져
  `(@08)`, `(ㅎ※#)` 같은 기호 쓰레기가 된 경우 괄호째 자동 제거합니다.
  한글 4자 이상이 들어있는 정상 괄호(역주 등)는 보존합니다.
- **AI 교정은 선택한 OCR 언어에 맞게 동작**합니다 — 한국어만 선택돼 있으면
  한글 자모 혼동(기봄→기쁨, 존제→존재 등)과 띄어쓰기를 교정하는 지침으로 전환됩니다.
- **✏️ 교정 규칙**: OCR 결과 아래의 "교정 규칙"을 펼치면, 반복적으로 잘못 인식되는
  글자를 `잘못=올바름` 형식으로 등록할 수 있습니다. OCR이 끝날 때마다 자동 적용되며,
  규칙은 저장되어 다음 실행에도 유지됩니다.
  - **🌐 공통 규칙** — 모든 작품에 적용되는 일반적인 혼동 글자
  - **📖 작품별 규칙** — 인명 등 작품 고유의 규칙. 처음 보는 `novelid`의 페이지를
    캡처하면 **작품 프로필이 자동 생성**되고(이름은 페이지 제목), 다음부터 그 작품을
    캡처하면 해당 규칙이 **자동 선택**됩니다. 여러 소설을 병행해도 규칙이 섞이지 않습니다.
  - **🔍 인명 오인식 분석** — OCR 결과에서 자주 나오는 한자 조합을 분석해,
    한 글자만 다른 변형들(예: 贺侍 4회 vs 贺尘 23회)을 찾아 교정 규칙 후보로
    제안합니다. 후보를 클릭하면 규칙 등록 + 즉시 적용됩니다.
  - **☁️ 자동 동기화** — 교정 규칙(공통 + 작품별 프로필)은 Chrome에 로그인한
    **Google 계정으로 자동 동기화**됩니다. 같은 계정으로 로그인하고 동기화가 켜진
    다른 기기에 이 확장(같은 폴더 내용)을 설치하면, 규칙이 자동으로 공유됩니다.
    manifest.json의 `key` 값 덕분에 모든 기기에서 확장 ID가 같아져 동기화가
    묶입니다 — **폴더를 수정 없이 그대로 복사**해서 설치하세요.
    (AI API 키는 보안상 동기화하지 않으므로 기기마다 입력해야 합니다)
  - **📤📥 규칙 내보내기/가져오기** — 공통 규칙과 모든 작품 프로필을 JSON 파일로
    저장하고, 다른 기기에서 가져올 수 있습니다. 가져오기는 기존 규칙을 지우지 않고
    병합합니다 (같은 작품은 새 규칙 줄만 추가, 처음 보는 작품은 새로 추가).
    Google 계정을 쓰지 않는 기기와 공유하거나 백업할 때 사용하세요.
- **🤖 AI 교정 (선택)**: OpenAI(ChatGPT) 또는 Anthropic(Claude) API 키를 넣으면
  OCR 결과를 AI가 문맥으로 교정합니다. 남아 있는 오인식을 잡는 가장 강력한
  수단입니다. 텍스트가 해당 회사 서버로 전송되고 API 사용료(챕터당 수 원 수준)가
  발생하므로 선택 기능입니다. 작품별 규칙에 등록된 인명은 "정답 표기"로 AI에게
  함께 전달됩니다.
- **🖼 AI 비전 OCR (선택, 최고 정확도)**: Tesseract 대신 **AI가 캡처 이미지를
  직접 판독**합니다. 중국어 간체 오인식이 사실상 사라지는 가장 확실한 방법입니다.
  같은 API 키를 사용하며 비용은 챕터당 수십 원 수준입니다. 이미지가 해당 회사
  서버로 전송된다는 점만 유의하세요.
- **📝 텍스트 바로 저장**: 起点(qdmm/qidian)처럼 본문이 일반 텍스트인 사이트에서는
  OCR 없이 **본문 섹션을 클릭하면 글자를 그대로 추출해 즉시 TXT로 저장**합니다
  (100% 정확). 파일명 규칙은 이미지 저장과 동일합니다.
